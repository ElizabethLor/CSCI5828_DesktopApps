History

History

  • IBM introduced a PC hardware platform on August 12, 1981.
    • The PC used a text mode commnad-line style operating systems known as Microsoft Disk Operating System (MS-DOS).
    • In the 1990's, the MS-DOS was replaced with the graphical Microsoft Windows Operating System.

History Con't

  • In 1991 and 1993, changes occurred that affected the world a great deal.
    • The World Wide Web (WWW) took off in 1991.
    • The Mosaic web broswer application was announced in 1993.

History Con't

  • Early applications were developed to be run from mainframe computers and accessed via low-tech terminal devices, but...
  • The increase power and availability of powerful desktop computers ushered in an era of standalone desktop applications that were run locally on the PC.

What is Desktop Applicaiton?

What is Desktop Applicaiton?

According to PC Mag, a desktop application is "an application that runs stand alone in a desktop or laptop computer", which is contrasted with a "web-based application, which requires the Web browser to run".

What is Desktop Applicaiton? Con't

  • Desktop applications runs on desktop platforms whereas mobile appliations run on smartphones and tablets.

Different Tools/Frameworks for Desktop Applications

Mac Toos/Framework: XCode

XCode

  • XCode was realease in 2003.
  • It is an integrated development environment (IDE).
  • It contains a suit of software development tools developed by Apple for developing software for OS X and iOS.
  • Tightly integrated with Cocoa and Cocoa Touch frameworks.
  • XCode supports:
    • C/C++/Objective-C/Objective-C++
    • Java
    • AppleScript
    • Python
    • Ruby
    • Rez
    • Swift

Mac Toos/Framework: Cocoa Framework

Cocoa Framework

  • An Apple's native object oriented application programming interface (API) for the OS X operating system.
  • It consists of the Foundation Kit, Application Kit and Core Data frameworks.
  • Apple provides development tools such as Xcode and Interface Builder to help Cocoa Frameworks develop applications.
  • Fun Fact: The framework was called "KidSim" before it was remaned to Cocoa.

Desktop vs. Web Application

Desktop vs. Web Application

  • "Software applicatino development begain with desktop applications, which could be used on standalone machines only.
  • Due to portability and better functions from usability point of view, web applications began to replace desktop applications.
  • Web applications development "is usually made on client server architecture and use a web browser as the client interface.

Desktop vs. Web Application

Features
Desktop Applications
Web Application
Rapid Developement
Designed to be a quick and easy solution to building graphical user interfaces (GUIs)
Was never designed for rapid development. The "correct" way to create websites is useing MVC (model, view, control)
Security
The program is running on the user's desktop; therefore this makes it harder for hackers to gain access to people's data
All of the user's data is stored online, taking just "one" hack to compromise "all" of the user(s) data

Desktop vs. Web Application Con't

Features
Desktop Applications
Web Application
Available Controls
There are a whole buffet of user interactive controls to choose from
No real controls except for the exception of HTML
Flexibility
Easy to write desktop applications that takes advantage of the user's hardware
Does not compare to the flexibility of desktop applications. Just stick with desktop application for your program

Desktop vs. Web Application Con't

Features
Desktop Applications
Web Application
Portability
Can be portable, but most are "not" and require manual installaiton from the user
Are "very" portable and will work with just about any computer with a decent web browser
